# 邮件来源分布图（Source Domain Map）功能说明与接口文档

## 概述

“邮件来源分布图（Source Domain Map）”用于分析该工具主要被用于哪些平台的账号注册/登录，从而帮助作者优化针对特定平台的正则提取模板。

可视化内容包含两部分：

- 词云图（Word Cloud）：基于“发件人域名（From Domain）”统计频次并可视化显示。
- 关系图（Force-Directed Graph / 关系网络）：展示“邮箱账号 ↔ 高频发信域名”的关联强度，用于判断每个账号主要在哪些平台活跃。

> 备注：当前版本为“最小可用版（MVP）”，不引入第三方图表库。词云使用“标签云”形式呈现；关系图使用简化版三列布局（账号列/边列/域名列）来表达关联。后续如需真正的力导向图，可接入 ECharts/D3。

---

## 数据来源与统计口径

### 当前数据源（已实现）

当前统计从本地 SQLite 数据库的 `temp_email_messages` 表读取：

- `email_address`：邮箱账号（节点之一）
- `from_address`：发件人地址（用于提取域名）
- `timestamp`：邮件时间戳（用于时间窗口过滤）

> 说明：该表是“临时邮箱邮件缓存表”。因此当前版本统计的是 **临时邮箱链路** 收到的邮件来源域名分布。

### 域名提取规则

从 `from_address` 提取域名：

- 支持 `Name <a@b.com>` 形式（优先取尖括号内的邮箱地址）
- 取 `@` 右侧作为域名，并转小写

## 页面入口（UI）

- 左侧导航：`分析 -> 来源分布`
- 页面 ID：`#page-source-map`

页面包含：

- 邮箱账号下拉：可选，仅统计指定账号
- 时间窗口下拉：近 30 天 / 近 90 天 / 全部
- 刷新按钮：拉取后端聚合接口并渲染

## 统一响应格式（本功能）

本功能接口返回 JSON：

- 成功：`success: true`
- 失败：`success: false`，并返回 `error`（沿用系统统一错误结构）

## 接口说明

### 1) 获取来源域名统计与关系图数据

**GET** `/api/analytics/source-domains`

#### 鉴权

- 需要登录态（Cookie Session）
- 未登录时：
  - 浏览器访问会跳转到 `/login`
  - 以 API 方式访问会返回 401（`AUTH_REQUIRED`）

#### Query 参数

| 参数 | 必填 | 示例 | 说明 |
|---|---:|---|---|
| `email` | 否 | `demo@local.test` | 只统计该邮箱账号 |
| `window` | 否 | `30d` / `90d` / `all` | 时间窗口，默认 `30d` |

#### 返回字段（成功）

```json
{
  "success": true,
  "email": "demo@local.test",
  "window": "all",
  "total_messages_scanned": 70,
  "top_domains": [
    {"domain": "github.com", "count": 30},
    {"domain": "accounts.google.com", "count": 18}
  ],
  "word_cloud": [
    {"domain": "github.com", "count": 30}
  ],
  "graph": {
    "accounts": ["demo@local.test"],
    "domains": [
      {"domain": "github.com", "count": 30}
    ],
    "edges": [
      {"account": "demo@local.test", "domain": "github.com", "count": 30}
    ]
  }
}
```

字段说明：

- `total_messages_scanned`：本次扫描的邮件行数（用于诊断/验证）
- `top_domains`：所有域名按 count 排序（最多返回前 50）
- `word_cloud`：词云展示数据（最多返回前 80）
- `graph.accounts`：参与图的账号列表（最多 12）
- `graph.domains`：参与图的域名列表（最多 18，只保留出现在 edges 中的域名）
- `graph.edges`：关联边（最多 80），count 越大表示该账号收到该域名邮件越多

#### 返回字段（失败示例）

```json
{
  "success": false,
  "error": {
    "code": "SOURCE_MAP_QUERY_FAILED",
    "message": "查询来源分布失败",
    "type": "DatabaseError",
    "status": 500,
    "details": "..."
  }
}
```

## 前端渲染说明

### 词云（标签云）

- 容器：`#sourceMapWordCloud`
- 单词节点：`.word-cloud-item`

已实现交互与可读性：

- Hover Tooltip：显示 `domain: xx封`
- 数量徽标：每个词右侧展示 `xx封`
- 视觉权重：字体大小采用幂次缩放（强化高频与低频差异）
- 少量旋转：约 10%-20% 的低权重词可竖排（MVP 中为固定规则）
- 防截断：容器允许 tooltip 溢出显示

### 关系图（简化版）

- 容器：`#sourceMapGraph`
- 三列布局：账号 / 关联边 / 域名

## 本地测试方法（无需上游临时邮箱服务）

由于临时邮箱上游可能出现 429 限流，建议使用“插入种子数据”来验证可视化效果。

### 1) 确认数据库路径

默认数据库路径来自 `outlook_web/config.py`：

- `DATABASE_PATH` 环境变量（可选）
- 默认值：`data/outlook_accounts.db`

### 2) 插入种子数据（示例）

插入多组不同域名、不同频次的数据到 `temp_email_messages`，然后在 UI 中刷新“来源分布”页面。

（具体命令可参考开发者在排障时使用的插入脚本/命令；若需要固化成脚本，可新增 `scripts/seed_source_map_test_data.py`。）

### 3) 验证点

- 访问页面：`来源分布`
- 选择 `window=all`
- 词云中高频域名明显更大
- Hover 提示显示正确数量
- Top 域名列表 count 与词云一致
- 关系图 edges 中 count 与域名统计一致

## 已知限制与后续扩展

1. 当前仅统计 `temp_email_messages`（临时邮箱链路）。
   - 若需要覆盖 Outlook/IMAP 正式账号，需要将邮件列表/详情落库或引入独立统计表。

2. 关系图为简化版布局。
   - 若要真正的力导向图，可引入 ECharts Graph / D3 Force 并使用 `graph.nodes + graph.edges` 结构直接渲染。
